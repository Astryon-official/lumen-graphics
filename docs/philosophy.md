# Lumen Philosophy

## Why Lumen Exists

Lumen was created with a simple goal:

> Make great graphics more accessible.

Rather than building a renderer for one specific game, Lumen is designed as a reusable graphics framework that can power visual enhancements across multiple games and desktop environments.

Every feature should contribute to a better visual experience while maintaining performance, flexibility, and simplicity.

---

# Core Principles

## Performance First

Visual improvements should never come at the cost of unnecessary performance loss.

Every feature should be designed to scale across different hardware where possible.

---

## Modular by Design

Lumen should be built as independent modules.

Rendering, lighting, post-processing, plugins, and game adapters should remain separate whenever practical.

This allows the framework to grow without becoming difficult to maintain.

---

## Engine Agnostic

Lumen should not be tied to one game or engine.

Game-specific code belongs inside adapters, not inside the rendering core.

---

## Extensible

New rendering techniques, plugins, and adapters should be easy to add without rewriting existing systems.

---

## Clean Architecture

Readable code is preferred over clever code.

Maintainability is more valuable than unnecessary complexity.

---

## Learn by Building

Lumen is both a graphics framework and a learning project.

The project should continuously evolve as new rendering techniques and technologies are explored.

---

# Long-Term Vision

Lumen aims to become a flexible graphics platform capable of enhancing games, desktop environments, and future Astryon technologies through modern rendering techniques.

The goal is not simply to add more visual effects.

The goal is to build a rendering framework that developers can understand, extend, and enjoy using.
